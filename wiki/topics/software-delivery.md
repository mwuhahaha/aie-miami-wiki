# Software Delivery

## Summary

Software delivery here means the part of the SDLC that has to absorb AI-generated code without letting review, architecture, and maintainability fall behind. The linked SDLC quality-gates talk frames the problem as a throughput mismatch: AI can produce implementation changes quickly, but teams still need explicit standards, automated checks, and review workflows that catch architectural drift before it becomes normal practice.

## Why It Matters

This topic matters outside the conference because AI coding tools change the bottleneck in engineering organizations. The hard part is no longer only generating code; it is deciding what code is acceptable, how those expectations are represented in machine-readable form, and where quality gates belong in planning, coding, review, and deployment. Qodo is relevant because it sits directly in that layer: AI-assisted code review and code-quality tooling for teams trying to make delivery faster without weakening the SDLC.
